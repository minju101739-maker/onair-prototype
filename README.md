# ON AIR — 캘린더 상태 기반 회의 조율 플랫폼

> 내 캘린더를 상태등처럼 읽어, 회의·업무 중인 시간은 켜고, 피해야 할 시간은 낮추고,
> 비어 있는 시간은 파랗게 열어 **회의가 잡히는 과정 전체(생성 → 동의 → 확정 → 기록)** 를 다루는 프로토타입.

**라이브 데모**: https://DEMO_URL_PLACEHOLDER/

- 프레임워크·빌드 없는 단일 `index.html` (Vanilla JS + CSS)
- 처음 접속하면 온보딩 투어가 자동 시작됩니다 — 상단 `?` 아이콘에서 화면 안내와 시나리오 가이드(회의 잡기 · 내 시간 지키기 · 초대에 답하기 · 회의 정리)를 언제든 다시 볼 수 있습니다

## 핵심 개념

| 개념 | 내용 |
|---|---|
| 회의 축은 투명, 개인 축은 비공개 | 목적·안건·참석·성립 조건은 전원에게 보이고, 불가 사유·개인 일정 상세는 본인 외 누구에게도 보이지 않음 |
| 제안 → 동의/불가 | 시간 투표가 아니라 순위화된 추천안에 대한 이분 응답. 필참 전원 동의 = 자동 확정 |
| 시간 판정 4분류 | ON AIR 확정(블루) · 응답 필요(대시) · 막은 시간(빗금) · 우선순위 낮음(연회색) — 후보 추천이 이 판정을 그대로 따름 |
| 기록 = 공유 정리본 | 참석자 중 한 명이 안건별 결론을 정리하면 전원이 같은 기록을 봄 — 별도 회의록 없음 |

## 화면

- **캘린더** — 주·일·월 뷰, 수명주기 4카드(이번 주 회의 · 응답할 초대 · 생성한 회의 · 정리할 회의), 좌측 응답 큐
- **구성원** — 조직도 트리 + 검색조건 바 + 업무정보 테이블, 선택 후 바로 회의 구성
- **회의 기록** — 기간·방식·목적 필터 + 결론·안건·정리자 컬럼, 정리 전/정리됨 두 상태
- **생성 위저드** — 목적 → 안건 → 참석(역할·필참/선택/공유만·현장/원격) → 시간 추천 → 발송

## 로컬 실행

빌드 과정이 없습니다. `index.html`을 브라우저로 열면 됩니다.

```bash
open index.html          # 또는
python3 -m http.server   # http://localhost:8000
```

데모 데이터는 시드 고정 PRNG로 생성되어 새로고침해도 항상 같은 상태를 봅니다(조직 134명 · 회의 기록 52건).
