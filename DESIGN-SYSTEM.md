# ON AIR — 디자인 시스템 (Data-first Compact)

> 제품 "ON AIR"의 UI 디자인 시스템. 업무용 데이터 테이블의 정렬감과 정보 밀도를 전 화면에 확장해, 캘린더·구성원·회의 기록·설정이 하나의 제품처럼 읽히는 인터페이스를 목표로 한다.

- 문서 버전: v0.16
- 작성일: 2026-06-29 · 개정: 2026-07-12
- 상위 문서: [CONCEPT.md](./CONCEPT.md) · 구현체: [index.html](./index.html)
- 변경 이력: v0.2 — 홈 화면 완성도 개선 반영. **§2.5 캘린더 카테고리 팔레트 신설**(고채도 원색 혼용 → 명도·채도 통일 저채도 세트), **§3.1 위계 원칙 신설**(화면당 대형 타이틀 1개), **§4.1 영역 구분 신설**(배경 위 카드 + 사이드바 헤어라인 섹션), **§5 FAB 규칙 추가**(주 액션은 우하단 FAB 하나), §6 아바타 팔레트 채도 하향. / **v0.3 — 심화**: §2.6 캘린더 레이어 구분, §2.7 시맨틱 상태색, §5.1 버튼 상태, §5.2 FAB 설명형 서브액션. / **v0.4 — 탈설명·탈색박스**: §8 신설. / **v0.5 — 용어 정리**: 제품 표기를 **내 기준**으로 통일. / **v0.6 — ON AIR 리브랜딩**. / **v0.7 — 구성·모바일·모달 UX 개편**. / **v0.8 — Data-first Compact 전환**: 큰 카드·큰 라운드·다층 그림자를 폐기하고 1px 구분선, 48–58px 데이터 행, 저대비 컬럼 헤더, 4–9px 라운드, 작은 상태 배지를 전 화면 공통 문법으로 채택. `캘린더 | 구성원 | 회의 기록`을 동급 1뎁스로 재구성하고, 월간 캘린더는 단일 월·단일 스크롤 소유자로 단순화. / **v0.9 — 조회·생성 통일**: 구성원과 회의 기록에 동일한 검색조건 바를 적용하고, 구성원 개인정보·상태값을 업무 정보로 정리했다. 회의 생성 진입은 단일 위저드로 통합하고 시간안 표기를 `추천안 / 대안`으로 통일했다. / **v0.10 — 정보 문법 통일**: 일시·장소·참석·동의 상태의 표기 규칙을 전 화면에 고정하고 목적 분류·기록자·작동 방식 설명처럼 판단에 불필요한 메타를 조회 UI에서 제거했다. / **v0.11 — 페이지 셸 통일**: 글로벌 헤더를 전체 폭에 고정하고 캘린더·구성원·회의 기록에 동일한 페이지 제목·설명·요약 구조와 타이포 토큰을 적용했다. 회의 기록 즐겨찾기와 넓은 일시 컬럼을 추가했다. / **v0.12 — 생성·제외 시간·기록 탐색 정비**: 생성 모달을 590px compact 위계로 낮추고 후속 기록을 46px 행으로 축소했다. `내 기준`을 `회의 제외 시간`으로 바꾸고 한 번/매주 반복 설정을 통합했으며, 기록 화면에 좌측 필터와 회의명·목적·장소 독립 컬럼을 추가했다. 수정·삭제는 아이콘 액션으로 통일했다. / **v0.13 — 탐색 패널·컬럼 원자화**: 구성원과 회의 기록의 좌측 패널을 같은 셸로 통합하고, 서로 다른 데이터 필드는 한 셀에 묶지 않는 원칙을 적용했다. 모든 표 데이터는 10.5px·동일 굵기·48px 행으로 정렬한다.
- **v0.14 — 생성 플로우 정비**: `계기`를 `목적`으로 통일하고 기타 목적, 4주 날짜 범위, 한 줄 후보 상태문, 공통 초대장과 안건 펼침, 캘린더 전용 FAB, 검색 셀렉트 공통 규격을 적용했다.
- **v0.15 — 회의 상세 통합**: 캘린더 일정·동의 요청·참석 요청·동의 현황·회의 기록 상세가 하나의 셸과 하나의 정보 카드를 사용한다. 공통 정보는 `회의명 → 목적 → 일시 → 장소 → 안건 → 참석자` 순서를 고정하고 상태와 액션만 모달 맥락에 따라 바꾼다.
- **v0.16 — 리듬·상태 디테일 정비**: 좌측 응답 패널과 우측 캘린더의 제목 시작선을 공통 토큰으로 맞추고 제목–부제 간격과 목록 행간을 고정했다. 사용자에게 보이는 ON AIR 상태는 파란색 `ON AIR 확정`으로 통일하며, 안건 제목 전체를 호버·포커스·클릭 가능한 상세 보기 영역으로 확장했다.

## 0. v0.16 적용 기준 — 충돌 시 이 절이 우선한다

아래 규칙은 하위 절에 남아 있는 이전 버전의 큰 카드·큰 라운드·그림자 수치보다 우선한다.

| 항목 | v0.16 기준 |
|---|---|
| 화면 구조 | `캘린더 / 구성원 / 회의 기록` 3개를 상단 동급 탭으로 둔다. 한 기능을 보조 모달에만 숨기지 않는다. |
| 글로벌 헤더 | `ON AIR`와 1뎁스 탭은 모든 화면에서 같은 좌표를 유지한다. 캘린더의 응답 패널은 헤더 아래 작업영역에서만 폭을 차지한다. |
| 페이지 헤더 | 세 화면 모두 `20px 제목 + 11px 설명 + 우측 요약`의 동일한 58px 헤더를 사용한다. 제목과 부제 사이는 6px, 설명 행간은 1.45로 고정한다. 별도 화면만 대형 제목을 쓰지 않는다. |
| 응답 패널 정렬 | 좌측 응답 패널은 페이지 좌우 여백 18px과 58px 제목 블록을 우측 캘린더와 공유한다. `응답할 회의`와 부제를 같은 시작선에 놓고 건수 배지는 제목 블록의 기준선에 맞춘다. |
| 검색 문법 | 구성원·회의 기록 모두 `검색조건 선택 + 검색어 + 조회 + 초기화` 한 줄을 쓴다. 테이블 안에 컬럼별 입력칸을 반복하지 않는다. |
| 즐겨찾기 | 구성원과 회의 기록 모두 상단 즐겨찾기 필터와 행 첫 컬럼의 별 버튼을 함께 제공한다. 별 클릭은 행 상세 열기와 분리한다. |
| 좌측 탐색 패널 | 구성원·회의 기록 모두 `204px 폭 / 46px 제목 / 스크롤 본문 / 34px 행 / 46px 하단 액션`의 동일 셸을 사용한다. 화면별로 별도 패널 디자인을 만들지 않는다. |
| 구성원 정보 | `즐겨찾기 / 성명 / 직무 / 직위 / 소속 / 이메일 / 출퇴근 시간`을 각각 독립 컬럼으로 조회한다. 재직 상태와 휴대폰 번호는 다루지 않는다. 조직도 루트는 `전체 구성원`이다. |
| 생성 흐름 | 모든 생성 진입은 같은 위저드로 연결한다. 시간 후보는 `추천안 / 대안 1 / 대안 2`로 표기하고, 과도하게 큰 제목·선택창·배지를 쓰지 않는다. |
| 목적 용어 | 사용자 화면에서는 `계기`를 사용하지 않고 모두 `목적`으로 표기한다. 목적 선택은 결정 필요·문제 발생·정보 공유·정기 회의·이전 회의 후속·기타 6종이다. `origin`은 내부 호환 필드로만 유지한다. |
| 기타 목적 | 기존 유형에 맞지 않으면 `기타`를 선택하고 목적과 안건을 직접 입력한다. 기타도 형식·시간·참석자·발송 단계에서 다른 목적과 동일하게 유지된다. |
| 생성 모달 밀도 | 기본 폭 590px, 모달 질문 16px, 선택 설명 9.5px, 후속 기록 46px 행을 기준으로 한다. 설명은 선택 결과에 직접 영향을 주는 경우만 남긴다. |
| 후보 탐색 기간 | 시작일·종료일은 실제 날짜 입력으로 제공하며 기본 2주, 최대 4주까지 탐색한다. 한 주의 요일 칩 두 줄로 범위를 제한하지 않는다. |
| 생성 상태문 | 오류가 아니면 경고 문구를 사용하지 않는다. `참석 가능한 시간 중 선택된 추천안입니다`처럼 선택 결과를 한 줄로 표시하고, 문장 끝 일부가 다음 줄로 떨어지는 설명은 축약한다. |
| 초대·발송 카드 | 발송 전 확인과 동의·참석 요청은 같은 초대장 컴포넌트를 사용한다. 추가 안건은 `외 N개 + 펼침 아이콘`으로 표시한다. 화살표만 작은 클릭 영역으로 두지 않고 첫 안건 제목 전체에 호버·키보드 포커스·클릭을 적용한다. 호버·포커스는 임시로 열고 클릭은 열린 상태를 고정한다. |
| 회의 상세 모달 | 캘린더 회의, 동의·참석 요청, 동의 현황, 회의 기록은 `560px / 11px 라운드 / 동일 헤더·닫기·본문 패딩`의 공통 셸을 쓴다. 본문은 `회의명 → 목적 → 일시 → 장소 → 안건 → 참석자` 순서를 바꾸지 않는다. 요청 상태는 카드 위, 응답·닫기·후속 액션은 카드 아래에만 둔다. 회의 기록은 같은 카드 아래에 결과 영역만 추가한다. |
| 생성 FAB | `+` 버튼과 스피드다이얼은 캘린더에서만 노출한다. 구성원·회의 기록에서는 숨기며, 화면 우하단 가장자리에서 40px 안쪽에 배치한다. |
| 검색 셀렉트 | 구성원·회의 기록 검색조건 셀렉트는 `116×36px`, 우측 12px 위치의 12px chevron을 공통 사용한다. |
| 내 일정(통합) | 회의 제외 시간을 별도 모달로 두지 않는다 — `내 일정` 모달 하나가 `한 번만 / 매주 반복` 탭과 2단계 선택(`우선순위만 낮춰요 / 막아둬요 — 절대 불가`)으로 전부 담당한다. 매주 반복 탭이 기존 반복 일정의 목록·수정·삭제까지 관리하며, 배지는 결과 그대로(`막음 / 후순위`)만 쓴다. |
| 날짜 선택 | 특정 날짜를 고르는 모든 입력(일정 추가·한 번만 제외 등)은 실제 날짜 입력(오늘~4주, 엔진 범위)을 쓴다. 이번 주 7일로 좁힌 요일 칩 선택은 금지 — 일정은 주 단위 개념이 아니다. 요일 칩은 `매주 반복`처럼 요일 자체가 개념일 때만 쓴다. |
| 편집 액션 | 수정·삭제는 연필·휴지통 아이콘과 접근성 라벨로 표시한다. 데이터 행에 `수정 / 삭제` 텍스트를 반복하지 않는다. |
| 일시 | 상세·기록은 반드시 `YYYY.MM.DD (요일) HH:MM–HH:MM`으로 표기한다. 연도나 시간이 빠진 축약형을 단독으로 쓰지 않는다. |
| 일시 컬럼 폭 | 회의 기록 데스크톱 일시 컬럼은 250px을 확보해 전체 일시를 한 줄로 표시한다. 중간 폭에서도 210px 이하로 줄이지 않는다. |
| 장소 | 오프라인은 실제 위치 한 줄, 온라인은 `온라인`만 쓴다. `온라인 · 화상 회의 · 링크 생성`처럼 당연한 정보를 반복하지 않는다. |
| 참석·동의 | 참석은 `아바타 + N명`, 동의는 `N/M`으로 표시한다. `내 응답`, `성립 조건`을 장문으로 다시 설명하지 않는다. |
| 기록 탐색 | 구성원과 동일하게 좌측 필터 패널 + 우측 데이터 표를 쓴다. 필터는 기간·방식·목적을 제공하고 검색과 함께 적용된다. |
| 기록 컬럼 | `즐겨찾기 / 일시 / 회의명 / 회의 목적 / 회의 장소 / 결론 / 안건 / 정리 / 참석자`를 각각 독립 컬럼으로 둔다. 설정 단계의 기록자는 노출하지 않는다. |
| 회의 정리 | 기록은 `정리 전 / 정리됨` 두 상태로 존재한다. 정리 전 기록은 구성 때 정한 목적·안건만 보이고 결론 셀은 비운다. 참석자 누구든 안건별 결론 한 줄로 정리할 수 있고, 한 명이 정리하면 전원이 같은 기록을 본다 — 별도 회의록 문서를 만들지 않는다. `정리` 컬럼에는 정리한 사람의 이름을 표기한다. |
| 정리 진입점 | 내가 참석한 정리 전 기록은 정리될 때까지 캘린더 홈 `회의 후 정리` 카드(큐)에 남는다. 정리 전 기록의 상세에는 항상 `회의 정리하기`가 있다 — 열람한 사람이 이어서 정리할 수 있어야 한다. 두 진입점(큐·상세)은 같은 정리 모달로 연결된다. 목록의 `정리` 컬럼은 상태(이름 또는 `정리 전`)만 균일하게 표기하고 셀 안에 액션 버튼을 섞지 않는다. |
| 기록 인명 | 기록의 참석자·정리자·담당자는 구성원 명부의 이름을 그대로 쓴다(성+이름). 구성원 화면에 없는 이름을 기록에 쓰지 않는다. 상세의 참석 칩은 이름만 표기하고 직급·소속은 호버 툴팁으로 내린다. |
| 기록 참석 상세 | 상세는 참석을 등급별 행(`필참 / 선택 / 공유만`)으로 나눠 각 인원과 인원수를 보여준다. 하이브리드 회의는 사람마다 `현장 / 원격`을 함께 적고, 대면·온라인은 형식 행으로 갈음한다. 공유만은 참석이 아니라 기록 수신이다. |
| 모달 정보 행 | 라벨과 값은 같은 글자 크기를 쓰고 위계는 선명도(색)로만 가른다 — 크기가 다르면 행 정렬이 어긋난다. |
| 데이터 셀 | 의미가 다른 두 필드를 한 셀에 묶지 않는다. 일시·회의명·결론도 강조 굵기를 쓰지 않고 다른 데이터와 같은 10.5px·550 weight를 사용한다. |
| 영역 구분 | 흰 단일 평면 + `1px #E1E5EA` 선. 카드 간 다층 그림자와 떠 있는 타일 배치를 쓰지 않는다. |

| 스크롤 | 한 영역에 스크롤 소유자 하나. 재렌더 후 현재 위치를 보존하고, 모달 배경과 본문을 동시에 스크롤하지 않는다. |

### 0.2 화면별 구조

- **캘린더**: 응답 큐(데스크톱만) + 통계 스트립 + 단일 캘린더 표면. 이벤트는 그림자 없는 셀 내부 블록이다.
- **구성원**: 공통 검색조건 행 + 조직 트리 + 즐겨찾기·업무정보 컬럼이 있는 구성원 테이블. 조직 범위와 검색 결과를 조합해 회의 참석자를 찾는다.
- **회의 기록**: 구성원 화면과 같은 공통 검색조건 행 + 같은 좌측 패널 셸의 기간·방식·목적 필터 + `일시 / 회의명 / 회의 목적 / 회의 장소 / 결론 / 안건 / 정리 / 참석자` 컬럼의 기록 테이블. 검색조건과 필터를 함께 적용해 과거 합의와 발언의 근거를 찾고, 행 클릭으로 안건별 상세를 연다. 기록 자체가 공유 정리본이다 — 참석자 중 한 명이 안건별 결론을 정리하면 전원이 같은 기록을 보고, 정리 전 기록은 목록에 `정리 전`으로 남아 누구든 이어서 정리할 수 있다.
- **설정·회의 제외 시간·생성**: 1px 선으로 묶인 행/섹션. 회의 제외 시간은 한 번/매주 반복 설정을 한 화면에서 제공하고, 생성 모달은 compact 행 중심으로 구성한다.

### 2.5 캘린더 카테고리 팔레트 (저채도 통일 세트)

캘린더처럼 여러 색이 동시에 놓이는 곳에서 고채도 원색(민트/보라/주황/핑크)을 혼용하면 화면이 촌스럽고 산만해진다. 카테고리 색은 **하나의 공식**으로 생성한 저채도 세트만 쓴다:

> **공식** — 배경 `bg`: 명도 L≈96 (거의 흰색에 가까운 은은한 틴트) · 텍스트 `fg`: 명도 L≈40, 채도 중간 (같은 색상環의 깊은 톤) · 도트 `dot`: 그 중간.

| 카테고리 | 토큰 | bg | fg | dot |
|---|---|---|---|---|
| 팀 정기 | `--cat-teal-*` | `#E9F5F0` | `#1D7D62` | `#27A583` |
| 1:1 · 리뷰 | `--cat-violet-*` | `#F0EFFB` | `#5D51B8` | `#8377E0` |
| 외부 · 채용 | `--cat-amber-*` | `#FAF1E5` | `#9A6B24` | `#E5A54E` |
| 개인 | `--cat-rose-*` | `#F9EFF1` | `#A15A6B` | `#DB93A4` |

- 이벤트 블록 = `bg` 틴트 면 + `fg` 텍스트. 그림자·보더 없음(플랫). 확정·오늘 등 **상태 강조는 블루 단독**(D3)으로만.
- 목적 유형 pill(발의 위저드)도 같은 토큰을 재사용한다 — 화면 어디서든 같은 의미는 같은 색.

### 2.6 캘린더 상태등 구분 (v0.6) — ON AIR의 핵심 은유

캘린더는 일정 종류보다 먼저 **시간 상태**를 보여준다. 같은 캘린더 데이터라도 화면에서는 아래 세 상태 중 하나로 투영된다:

| 상태 | 의미 | 시각 언어 |
|---|---|---|
| **ON AIR** | 회의·업무 일정이 진행 중인 시간 | 일정 블록 + 붉은 점등(`--air`). 이 시간에는 새 회의가 잡히지 않음 |
| **피함** | 내 기준·오늘 상태상 피하고 싶은 시간 | 그레이/경고 틴트. 가능한 시간이지만 추천 순위가 낮음 |
| **비어 있음** | 일정이 없는 조율 가능 시간 | 파란 면(`--free-soft`). 후보 산출의 기본 출발점 |

그 위에 보조 레이어로 출처를 표시한다. **ON AIR에서 확정**한 회의는 블루 틴트로, 연동 일정은 카테고리 저채도 틴트(§2.5)로, 개인 일정은 흰 배경 + 점선 테두리로 구분한다. 캘린더 상단 범례는 `비어 있음 / ON AIR / 확정 / 개인 / 내 기준`을 상시 명명한다. 내 기준 옆에는 "나만 봄" 문구 고정.

### 2.7 시맨틱 상태색 (v0.3)
`--warn #B25E09/#FFF3E0`(경고 박스·최악 조건), `--danger #C0392B/#FDECEE`(비동의·삭제) — 카테고리 앰버와 분리해, **경고는 시맨틱 토큰만** 쓴다.

### 3.1 위계 원칙 — 화면당 대형 타이틀은 하나

크기·굵기가 "전부 크고 굵으면" 위계는 없는 것과 같다. 홈 화면 기준 서열:

| 서열 | 요소 | 스펙 |
|---|---|---|
| 1 | 화면 타이틀 ("내 회의") | 20px / 800 — **화면에서 유일한 대형 타이틀** |
| 2 | 인사말 중 이름, 요일 날짜 숫자 | 15px / 700–800 (오늘 날짜만 블루 원형 배지) |
| 3 | 목록 아이템 제목 (할 일·일정) | 13px / 700, `--ink` |
| 4 | 보조 설명·시간·장소 | 11–12px / 400–500, `--faint` |
| 5 | 섹션 라벨 (사이드바) | 11px / 700, uppercase, `--faintest` |

- 인사말·기간(7/1–7/7)·부서명 같은 맥락 정보는 **강조하지 않는다** — 이름만 800, 나머지는 `--muted`/`--faintest`.
- 요일 헤더의 날짜 숫자는 화면 타이틀보다 항상 작게. "오늘"은 크기가 아니라 **블루 원 하나**로 구분한다.

## 4. 스페이싱 · 형태 · 그림자

- **라운드**: `--r: 20px`(카드), `--r-sm: 14px`(버튼·입력·내부 요소), `--r-xs: 10px`, pill은 `999px`.
- **카드 패딩**: 헤더 `24px 26px 0`, 바디 `22px 26px 26px`.
- **그림자**:
  - `--sh: 0 1px 3px rgba(0,27,55,.04), 0 6px 20px rgba(0,27,55,.06)` — 카드 기본(부드럽고 낮게).
  - `--sh-lg: 0 16px 48px rgba(0,27,55,.18)` — 토스트 등 떠 있는 요소.
- 콘텐츠 최대 폭 `1240px`, 좌우 패딩 `clamp(18px, 3.5vw, 40px)`.

### 4.1 영역 구분 — 배경 위에 뜨는 카드

"어디까지가 한 영역인가"는 보더가 아니라 **배경 대비 + 여백**으로 답한다:

- **메인 콘텐츠(캘린더)**: 페이지 배경(`--bg`) 위에 흰 카드(라운드 16px + 낮은 그림자)로 띄운다. 카드 밖 16–20px 여백이 곧 영역의 경계다. (`.cal-wrap` / `.cal-card`)
- **사이드바**: 흰 면 + 오른쪽 1px `--line`. 내부 섹션(할 일 / 내 일정 / 다가오는 일정 / 회의 기록)은 **헤어라인(`--line2`) + uppercase 라벨 + 상단 여백 16px** 세트로 반복 구분한다. (`.side-sec`)
- **탑바**: 흰 면 + 하단 1px `--line`. 인사말(좌) / 상태 pill·알림(우)의 2존.
- 카드 안의 내부 구분은 더 약한 선(`--line2`)만 — 바깥 경계보다 안 경계가 진하면 안 된다.

## 5. 컴포넌트 규칙

- **회의 생성 진입점은 두 축까지만**: 회의 생성은 사이드바 시작 카드와 우하단 FAB 스피드다이얼의 `회의 바로 생성`으로만 둔다. 상단에는 회의 생성 버튼을 두지 않는다. FAB는 `회의 바로 생성 / 내 일정 추가` 두 액션 버전이 기본이다.
- **바로 생성 시트**: `30분 싱크 / 1:1 / 결정 회의` 같은 템플릿과 날짜·시간·형식만 먼저 묻고, 기존 위저드의 참석자 구성 단계로 이어진다. 빠른 길은 긴 길의 우회가 아니라 **앞단을 채운 같은 객체**다.
- **설정 운영 화면**: 설정은 단순 목록 모달이 아니라 ON AIR 상태, 내 기준, 캘린더 연동, 데이터 관리가 카드형으로 묶인 2뎁스 화면이다. 법적/프라이버시 설명은 이 화면 안에서만 허용한다.
- **모달 헤더**: 상단 바는 조용한 내비게이션 존 — 타이틀은 **12px/850 uppercase 라벨**, 위계의 정점은 본문의 **단계 질문/대상 제목(`.mc-h h3`, 21px/850)** 하나다. 모달 표면은 28px 라운드, 약한 보더, 큰 그림자로 배경과 분리한다.
- **상태 카드(투표 선두 등)**: 컬러 보더로 감싸지 않고 **soft 배경(`--blue-s`) + 얇은 라인(`--blue-l`)** 으로. 게이지 바는 3px, 상태 pill은 블루 단독.
- **버튼 `.btn`**: 15px/700, 패딩 `14px 22px`, 라운드 14px, press 시 1px 하강.
  - `.btn-primary` = 블루 채움(hover `--blue-deep`), `.btn-ghost` = 패널2 배경 + 라인.
- **입력 `.input/.select`**: 패널2 배경, 포커스 시 블루 2px 아웃라인 + 흰 배경.
- **스텝퍼 `.step`**: 활성/완료는 `--blue-soft` 배경 + 블루 텍스트, 활성 번호는 블루 채움.
- **후보 카드 `.cand`**: 선택 시 블루 테두리 + `--blue-soft`. 등급 배지 `grade`: **S=블루, A=민트, B=그레이**.
- **투표 `.vopt`**: 선두는 블루 테두리 + 글로우, 게이지 바는 민트(선두는 블루). 득표 수는 대형 모노.
- **pill**: 기본 무채색, `.cov`(참석 커버리지)=블루-소프트, `.ok`(편안)=민트-소프트.
- **프라이버시 노트 `.priv-note`**: 블루-소프트 배경 + 방패 아이콘. 경고색을 쓰지 않는 것이 핵심.

### 7.2 캘린더 스크롤 · 주간 구성

- 주간 뷰는 **연속 7일**(월–일 또는 기간 표기와 일치하는 7일). 업무일만 추려 5열로 보여주면 "일주일"이라는 기간 감각이 깨진다. 주말 열은 이벤트 없이 `--panel2` 톤으로 조율 대상이 아님만 표시.
- 표시 범위는 07–22시 **세로 스크롤** — 업무시간(9–17)은 조율 엔진의 탐색 범위일 뿐 화면을 가두는 틀이 아니다.
- 최초 스크롤 위치는 09:00, 이후 사용자의 위치를 재렌더에도 보존.
- 시간당 64px 고정 리듬. %-기반 이벤트 블록과 시간 라벨은 항상 같은 축척(열 박스 `height:max-content`)을 공유해야 한다.
- **내 기준 오버레이**: 내 기준과 오늘 상태가 캘린더에 그대로 보인다 — 피함 = 그레이/경고 틴트, 좋음 = 민트 저채도 틴트(`rgba(15,201,166,.09)`). 이벤트 아래 레이어, 클릭 통과, 헤더의 "내 기준" 토글로 on/off. 일정이 없는 영역은 기본적으로 파란 "비어 있음" 면이고, 그 위에 피함/좋음이 덮인다. 기준 모달·캘린더 존·후보 순위는 **같은 데이터(MY_PREFS + TODAY_CONTEXTS)** 를 쓴다 — 설정과 화면이 어긋나면 신뢰가 깨진다.

### 7.3 1뎁스 원칙 — 메인에 두는 것과 두지 않는 것

메인 화면 1뎁스는 **"오늘 내가 행동할 것"**(내 차례)과 **"내 시간·회의의 상태"**(캘린더·조율 현황·분류·연동)만 갖는다.

**세 축을 섞지 않는다 — 알림 ≠ 내 차례 ≠ 조율 현황. 판별 기준은 "시간축"과 "공이 누구 코트에 있나":**

| 축 | 정의 | 시점 | 공 | 형태 |
|---|---|---|---|---|
| 알림 | 지나간 **사건의 통지** ("확정됐습니다", "○○ 수락") | 과거 | — | 벨 피드 (읽으면 끝, 일회성) |
| 내 차례 | 내가 풀어야 진행되는 **블로커** | 현재 | **내 코트** | 행동 큐 — 서브라인에 "안 하면 무엇이 멈추는지" 명시 |
| 조율 현황 | 내 몫은 끝, **남의 응답을 기다리는** 회의 | 현재 | **상대 코트** | 모니터링 행 — 동의 게이지(n/m). 액션 아님 |

**핵심 규칙 — 한 회의는 내 차례와 조율 현황 중 한 곳에만 있는다 (상호 배타).** 판별은 "지금 공이 누구 코트에 있나": 내 응답·설정이 남았으면 → 내 차례. 내 몫이 끝나 남을 기다리면 → 조율 현황. 내가 응답하는 순간 그 회의는 내 차례에서 조율 현황으로 이동한다. 같은 회의를 두 곳에 동시에 두면 "왜 겹치지?"라는 혼란이 생긴다.

- 알림은 **과거 시제**만 — "응답하세요"(미래 행동)는 알림이 아니라 내 차례다. 알림에 행동 요청을 넣으면 내 차례와 겹친다.
- "초대받음 · 마감 D-1" 같은 통지형 문구를 할 일에 쓰면 알림과 구분이 사라진다 — 내 차례 서브라인은 결과("미응답이면 회의 미성립")를, 조율 현황 우측은 진행도(동의 3/4)를 말한다.

- **회의 기록 = 독립 1뎁스**: 캘린더·구성원과 동급인 상단 탭으로 둔다. 목록은 `일시 / 회의명 / 회의 목적 / 회의 장소 / 결론 / 안건 / 참석자` 테이블이며, 행을 눌렀을 때만 안건별 기록 상세를 모달로 연다.
- **기록자·목적 분류 같은 내부 메타는 조회 UI 금지**: 목록과 상세 모두 제목·정확한 일시·장소·참석·안건·결론·액션만 보여준다.
- 같은 정보를 두 곳에 두지 않는다 — 캘린더에 있는 "다가오는 일정"을 사이드바에 중복 나열하지 않는 것과 같은 원칙.

## 8. 절대 하지 않는 것 (금지) 🚫

> 이 항목들은 "지양"이 아니라 **금지**다. 한 번도 쓰지 않는다.

### 8.1 컬러 좌측 보더 액센트 선 — **절대 금지**
카드·타일의 한 변(특히 왼쪽)에 두꺼운 색 선을 그어 강조하는 패턴.

```css
/* 🚫 금지 — 절대 쓰지 않는다 */
.tile { border-left: 4px solid var(--blue); }
.card { border-top: 3px solid var(--mint); }
```

- 촌스럽고(올드한 대시보드/관리자 페이지 느낌), 토스 미감과 정면으로 충돌한다.
- 색 선 하나로 영역을 "표시"하려는 발상 자체를 버린다.

**대신 이렇게 강조한다:**
- 카드 전체 배경을 **soft 톤**(`--blue-soft` / `--mint-soft`)으로.
- 상태는 **pill**, **텍스트 색**, **아이콘**으로.
- 위계는 **여백 + 부드러운 그림자(`--sh`)**로. (원칙 D2·D4)

### 8.2 함께 금지하는 것
- **두꺼운 컬러 테두리**로 카드를 감싸기 (강조는 배경/그림자로).
- **여러 강조색 동시 사용** — 강조는 블루 하나, 보조는 민트까지(원칙 D3 위반 금지).
- **하드 드롭섀도우**(짙고 각진 그림자), 짙은 1px 풀-블랙 보더.
- **그라데이션 남발** — 면 채움은 단색 우선.
- **고채도 원색을 식별 색으로 혼용** — 카테고리·아바타 등 다색 공존 영역은 §2.5 공식의 저채도 세트만 (원칙 D6).
- **화면당 두 개 이상의 대형 타이틀 / 파란 대형 CTA 버튼 중복** — 위계의 정점은 하나(§3.1), 주 액션은 FAB 하나(§5).

## 7. "준비 중" 금지 원칙 (v0.3)

> **모든 인터랙션은 (a) 실제 상태를 바꾸거나, (b) 왜 안 되는지 구체적 사유를 말해야 한다. "준비 중입니다"는 금지.**

- 참석 응답 → 실제 수락/불가 모달(초대장·역할·준비물 표시, 응답 시 할 일에서 제거).
- 회의실 배정 → 실제 빈 방 목록에서 선택·예약 반영.
- 기준 추가 → 인라인 폼(이름·피함/좋음·요일·시간·강도), 저장 즉시 캘린더 존 반영.
- 근무시간 수정 → 칩 편집, 저장 시 기준 앵커 재환산 안내.
- 데이터 내보내기 → **실제 JSON 파일 다운로드**. 삭제 요청 → 구체 정책 안내(30일 내 파기·메일 통보).
- 캘린더 연동 → OAuth 스코프를 보여주는 동의 모달(Google=구조적/MS=정책적) 후 상태 전환. 연동 불가 소스는 사유를 명시(iCloud→CalDAV+앱 암호 안내).

### 5.1 버튼 상태 (v0.3, §5 보충)
`hover` 배경 변화(즉시) → `active` 한 단계 깊은 색(`--blue-p`) → `focus-visible` 2px 블루 아웃라인. 애니메이션·transform 없음(무모션 원칙).

### 5.2 FAB 서브액션 (v0.3, §5 보충)
스피드다이얼 항목은 **아이콘 타일 + 제목 + 행위 설명 한 줄**("왜 이 둘이 다른 행위인지"를 문장으로): 회의 바로 생성 — *템플릿과 시간만 잡고 참석자 구성으로 이동* / 내 일정 추가 — *조율 없이 내 캘린더에만 기록*. 열리면 스크림으로 배경을 가라앉히고, Esc·스크림 클릭으로 닫는다.

## 8. 탈설명 · 탈색박스 원칙 (v0.4)

1. **설명문 금지.** UI 안에 "이것은 ~입니다 / ~하면 사라집니다" 류의 안내 문단을 두지 않는다. 좋은 구조는 설명 없이 읽힌다 — 설명이 필요하면 구조를 고친다. (프라이버시 고지 등 법적 필수만 예외, 설정 화면에 한정)
2. **색 박스에 텍스트를 넣지 않는다.** 색 배경 + 텍스트 조합(틴트 칩·컬러 안내 박스)은 촌스럽다. 색은 **점(dot)·스와치·좌측 띠**로만 쓰고 텍스트는 플레인으로: `● 필참`, `● 결정`. 경고도 배경 없이 워닝 색 텍스트 + 좌측 띠.
3. **색 박스 + 아이콘은 허용.** 텍스트가 없는 컬러 타일(아이콘 배경)은 식별에 유효하다 — 금지 대상은 어디까지나 "색 배경 위의 텍스트"다.
4. **부제는 한 호흡.** 목록 부제는 잘리지 않는 길이(10자 내외)로 — 상태 하나만 말한다("이월 2건", "마감 D-1"). 설명은 클릭한 화면이 한다.
